# MatterVibe 多设备同步指南（Mac + Windows）

> 本版（0.9.9.3）提供**同步地基与手动同步包**：每台设备各存一份完整本地数据，
> 通过一个「同步包」文件在设备间合并。下一步将做 WebDAV/NAS 自动同步，届时这套
> 操作会变成后台自动完成。先理解原理，自动化只是把"手动拷文件"这步交给程序。

## 核心原理（务必先懂这一点）

MatterVibe 的数据是一个 SQLite 单文件数据库，存在每台设备自己的本地：

- macOS：`~/Library/Application Support/MatterVibe/law_practice.db`
- Windows：`%AppData%\MatterVibe\law_practice.db`

**绝对不要把这个 .db 文件直接放进网盘同步文件夹（坚果云/iCloud/OneDrive）让两台电脑共用。**
SQLite 靠文件锁保证写入完整，网盘不懂数据库结构，会在写入中途抓取文件，
轻则产生"冲突副本"丢数据，重则把数据库写坏。这是同步最大的坑。

正确方式：**每台设备各自保留本地数据库（保证断网秒开），设备之间只交换"操作记录"
（谁在何时做了什么），由程序合并。** 这就是本版的「同步包」。

## 当前用法：手动同步包

1. 在设备 A（如 Mac）：设置 → 设备同步… → 确定（导出同步包），得到一个
   `mattervibe-sync-xxxx.json` 文件
2. 把这个 json 传到设备 B（如 Windows）：U盘、隔空投送、微信传文件、或任意网盘均可
3. 在设备 B：设置 → 设备同步… → 取消 → 确认"导入并合并"，选择那个 json
4. 设备 B 会把设备 A 的变更合并进来（已存在的自动跳过，导入前自动备份留底）

反方向同理。每台设备有唯一「设备号」，合并按操作记录去重，重复导入不会出错。

## 方案一：用坚果云实现"半自动"同步（推荐入门）

坚果云支持把某个文件夹设为同步盘。思路：**把"同步包"放进坚果云同步文件夹**（注意：
是同步包 json，**不是** law_practice.db），让两台设备自动收发同步包，你只需在软件里点"导入合并"。

步骤：

1. 两台设备都安装坚果云，登录同一账号，新建一个同步文件夹，例如 `MatterVibe同步/`
2. 每次在一台设备改完案件，导出同步包，保存到这个文件夹（坚果云会自动上传）
3. 另一台设备打开后，坚果云已把 json 同步下来，在软件里「导入并合并」即可

**重要的保密提醒：** 同步包里含案件信息，经过坚果云服务器。在涉及敏感案件时，
建议把同步包先用密码压缩（如 7-Zip 加密压缩）再放进坚果云，对方解压后再导入；
或等待下一版的端到端加密自动同步。

坚果云免费版每月有上传流量限制，但同步包是纯文本、体积很小，通常完全够用。

## 方案二：用群晖 NAS 实现真正的自动同步（最佳，数据不出家门）

如果你有群晖（Synology）NAS，这是保密性最好的方案——数据中转始终在你自己的 NAS 上，
不经任何第三方公司服务器。

NAS 端准备（一次性）：

1. 群晖控制面板 → 文件服务 → 启用 **WebDAV**（记下端口，HTTPS 默认 5006）
2. 新建一个共享文件夹，例如 `mattervibe`，给你的账号读写权限
3. 确保 NAS 可被你的设备访问：家里同一局域网可直接用内网 IP；在外需要开启
   群晖的 QuickConnect 或做端口转发 + DDNS

当前版本下的用法（手动，但中转在自己 NAS）：

1. 把 NAS 的 WebDAV 文件夹挂载为网络驱动器（Mac：访达 → 前往 → 连接服务器
   `https://你的NAS地址:5006`；Windows：此电脑 → 映射网络驱动器）
2. 导出的同步包直接存到这个挂载目录，另一台设备从同一目录读取并导入合并

下一版（WebDAV 自动同步）做好后：你只需在软件里填一次 NAS 的 WebDAV 地址、账号、
密码和一个加密口令，软件就会自动定期把本机操作记录加密上传、把别的设备的记录拉下来
合并，全程无需手动导出导入。数据在传输和存储时都是密文，连你的 NAS 管理员都看不到内容。

## 关于团队多人使用（已预留）

本版的操作记录已经带「设备号」和「操作者」两个维度，正是为将来团队协作预留的。
当团队同事接入时，每个人是一个"操作者"，同步合并时能区分谁做了什么，
不需要推倒重来。团队场景下，中转建议用 NAS 或一台小服务器集中存放，
并为每人配独立账号——这部分会在多人协作版本中完善。

## 路线小结

- **现在（0.9.9.3）**：手动同步包，机制已通；可配坚果云/NAS 做半自动

- **再下一步**：团队多人协作（多操作者、集中中转、权限）
