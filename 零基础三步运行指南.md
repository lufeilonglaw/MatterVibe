# MatterVibe 1.4.1 · Mac 零基础三步运行指南

一个完全本地运行的、以案件为中心的看板客户端。所有数据保存在你 Mac 本机的
SQLite3 单文件数据库 `law_practice.db` 中，不联网、不上传、关掉即存。

---

## 第〇步（仅一次）：确认电脑上有 Node.js

打开「终端」（聚焦搜索 ⌘+空格，输入 Terminal 回车），粘贴运行：

```bash
node -v
```

显示形如 `v20.x.x` 的版本号即可跳到第一步；提示找不到命令，
则去 https://nodejs.org 下载 macOS 安装包（LTS 版）双击安装。

---

## 第一步：进入项目文件夹

在终端输入 `cd `（cd 后面有个空格），把 `matterflow` 文件夹直接拖进终端窗口，回车。

## 第二步：安装依赖（仅第一次需要，约 1～2 分钟）

```bash
npm install
```

## 第三步：启动客户端

```bash
npm start
```

以后每次使用，只需重复第一步 + 第三步。

---

## ⚠️ 从旧版本升级：如何保住现有数据

你的全部案件、模板和设置都在**一个文件**里，升级 = 先把这个文件留好。

**1. 找到旧数据文件。** 按你旧版的使用方式，在下面两处之一：

```
~/Library/Application Support/matterflow/law_practice.db          ← npm start 方式
~/Library/Application Support/MatterVibe/law_practice.db          ← 2.2 起 .app 方式
~/Library/Application Support/魔流 MatterVibe/law_practice.db      ← 2.1 版 .app 方式（旧）
```

注意：2.2 起应用更名为 MatterVibe，**打包版的数据目录随之变化**。
若你此前用的是 2.1 的 .app，请把旧目录里的 `law_practice.db`
复制到新目录 `~/Library/Application Support/MatterVibe/`（首次启动新版后该目录会自动生成）。

终端里可直接打开目录确认：`open ~/Library/Application\ Support/`

**2. 复制备份一份**（拖到桌面即可），以防万一。

**3. 用新版 `matterflow` 文件夹覆盖旧文件夹**（或放到新位置），
重新执行 `npm install`。**不要动** Application Support 里的数据文件，
新版启动时会自动读取它，并自动补建 2.1 新增的表（备份表、设置表），
旧案件、旧模板原样保留，无需任何手工迁移。

**4. 如果你换了启动方式**（例如旧版用 `npm start`、新版打包成了 .app），
两个目录的数据不互通——把第 1 步那个 `law_practice.db` 复制到另一个目录即可。

## 想要一个能双击的独立 .app？（可选）

```bash
npm run pack
```

完成后在 `dist/MatterVibe-darwin-arm64/` 里生成 `MatterVibe.app`，拖进「应用程序」即可双击使用。
（Intel 芯片的 Mac 把 package.json 里 pack 命令中的 `arm64` 改成 `x64`。）
首次双击若提示「无法验证开发者」：右键 .app → 打开 → 再点「打开」，仅需确认这一次。

## 1.3.0 更新内容

- **坚果云自动同步（自动传输）**：在「设置 → 设备同步 → 坚果云自动同步」里配置一次，之后
  打开软件就自动把本机改动加密上传坚果云、并拉取其他设备的改动合并。也可手动「立即同步」，
  或设为每 10/30/60 分钟定时。配置见下方《坚果云同步配置》
- 端到端加密：所有上传到坚果云的数据用 AES-256-GCM 加密，**加密口令只在本机、绝不上传**，
  云端只存密文。可「导出口令备份」妥善保管；换新设备时输入相同口令即可解密
- 手动同步包方式（导出/导入 .json）仍然保留，可与自动同步并用

### 坚果云同步配置（一次性）

1. 登录坚果云网页端 → 右上角账户 → 安全选项 → 第三方应用管理 → 添加应用密码
   （名称随意，如 MatterVibe），生成一个**应用密码**（这不是你的登录密码）
2. 在 MatterVibe：设置 → 设备同步 → 坚果云自动同步，填：
   - 服务器地址：保持默认 `https://dav.jianguoyun.com/dav/`
   - 坚果云账户：你的注册邮箱
   - 应用密码：上一步生成的那个
   - 加密口令：**你自己设定**一个口令（用于加密案件数据），输两遍
3. 点「测试连接」确认无误，再「保存并启用」；建议随后「导出口令备份」保存到密码管理器
4. 在另一台设备重复 2–3，**加密口令必须填与第一台完全相同的那个**，否则无法解密云端数据

> 提醒：加密口令一旦遗忘，云端已加密数据将无法恢复（本地数据不受影响）。务必妥善保管。

### 关于"数据库直接放网盘"——不要这样做

不要把 `law_practice.db` 直接放进坚果云/iCloud/OneDrive 让两台电脑共用。SQLite
靠文件锁保证写入完整，网盘会在写入中途抓取文件，可能产生冲突副本丢数据、甚至写坏
数据库。正确方式是用上面的「同步包」机制（设备各存本地、只交换操作记录）。

### 升级到本版的说明（重要）

本版改名后数据目录从 `~/Library/Application Support/MatterFlow/` 变为
`~/Library/Application Support/MatterVibe/`。**首次启动会自动迁移**：把旧目录的
`law_practice.db` 与 `backups/` 复制到新目录。旧目录不会被删除，万一需要可手动找回。
建议升级前照例在「案件管理 → 备份与恢复」手动备份一次。

**如果你升级后发现数据"消失"了**（早期 0.9.9 存在一个迁移失效的问题）：你的数据并没有丢，
仍完好地保存在旧目录里。按以下步骤手动找回（一分钟）：

1. 完全退出 MatterVibe（菜单 → 退出，或 Cmd+Q，不要只关窗口）
2. 打开「访达」，按 Shift+Cmd+G，前往：`~/Library/Application Support/MatterFlow`
3. 把里面的 `law_practice.db` 复制（Cmd+C）
4. 再前往：`~/Library/Application Support/MatterVibe`（若不存在就新建一个同名文件夹）
5. 粘贴（Cmd+V）。如旧目录还有 `backups` 文件夹，也一并复制过去
6. 重新打开 MatterVibe，数据即恢复

本版已修复该迁移问题：今后从旧目录升级会自动完成上述迁移，无需手动操作。

| 克隆案件 | 设置（齿轮）→ 克隆当前案件 |
| 编辑模板 | 设置（齿轮）→ 模板编辑；左侧「＋ 新建模板」可增加自定义模板 |
